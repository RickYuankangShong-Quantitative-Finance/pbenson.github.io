This began as a collection of my sketches in Processing and p5. That stuff is still here. But the focus now is on my work with students in the Quantitative Finance and Risk Management program at the University of Michigan.

QFRM Bootcamp agenda

The week before QFRM students begin their fall semester, we run a "bootcamp" to prepare resumes, build a University web page and personal website, and review/learn some financial concepts while learning some Python. Below is the agenda.

  1. Installing Workshop tools, Anaconda, Atom, GitHub client. We assume you have a word processor for creating a resume, such as Google doc or MS Word.
  2. Overview of your online employment portfolio
    1. Your U of M home page.
    2. resume
    3. LinkedIn page
    4. personal webpage

QFRM student websites

Students in the QFRM program at the U of M create personal websites to describe and provide evidence of the quality of their work. You can create a hosted site easily using Google Sites or Wordpress.org, but these solutions are blocked in China, which is undesirable if you intend to apply for jobs there. You can test the accessibilitiy of a website via blockedinchina.net.

GitHub Pages sites are free, and are not blocked. You can roll your own websites, or search online for a template, or you can start with a template from below. These were orginally inspired by CSS Zen Garden. All of the templates have exactly the same HTML, which is customized by the student using a text editor like Atom. As long as the student does not change class names in the HTML document, they can easily change the look of their site by swapping a new styles folder.

  2. Zen 214
  3. Zen 220

QFRM Python

Through the development of powerful libraries such as numpy, pandas, scipy and others, Python has become a popular tool on Wall Street. In the QFRM bootcamp, we practice using these Python to solve math problems, and use the libraries to solve financial problems.

Getting started

  1. Install Anaconda if you don't already have it. We prefer the 2.7 edition of Python, and recommend the graphical installer.
  2. Follow step one of the Conda 30 minute test drive
  3. Follow step four of the Conda 30 minute test drive.
  4. Launch Jupyter Notebook. You may want to create a special directory for your Jupyter notebook files which you can maintain with GitHub.
