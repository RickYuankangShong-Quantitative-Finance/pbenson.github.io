Getting Started

There are many C++ IDEs to choose from. Visual Studio is preferred on Windows, and Xcode on Mac OS X. Both are free. There is less consensus about preferences on Linux. We assume, however, that you are using Windows or Mac.

Please note that the install can take a long time on either platform. Get the install completed before class, and ask for help if needed, as there won't be time in class to correct significant installation issues.

  1. Download and install the community edition of Visual Studio (Windows) or install Xcode from the Mac App Store.
  2. For additional help getting started with C++ in Xcode, see Jesse Daugherty and Chris Wendel's site, and the Xcode Notes for EECS 381. For Visual Studio, see EECS 183 notes Getting Started with Visual Studio.
  3. Let's create our first program. The purpose here is to get introduced to creating a project in your IDE (Xcode or VS), compiling and running it successfully, and then committing it to GitHub. Launch your IDE, and select File->New Project, and create a Command Line application named HelloWorld. This creates a file called main.cpp. Replace the contents of main.cpp with the source code below.
    Xcode
    #include <iostream>
int main(int argc, const char * argv[]) {
    std::cout << "Hello, World!\n";
    return 0;
}
    Visual Studio
    #include "stdafx.h"
#include <iostream>

int main(int argc, const char * argv[]) {
	std::cout << "Hello, World!\n";
	system("pause");
	return 0;
}
    Note that VS has an additional include file called stdafx.h. This should be included in the main.cpp file of every C++ program you write (it's for faster compiling of Windows apps). You will also want to include the system("pause"); snippet (unless you have an event loop). Otherwise, your application will close before you can see the results of running your program.
  4. To do serious math, you will want the boost libraries as well. For Windows, see boost installation and configuration in VisualStudio. On the Mac, follow the directions of the most popular answer here on stackexchange, which involves first installing homebrew.
